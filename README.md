# FBA 运费重量对比

一个基于 `FBA运费模板.xlsx` 制作的静态运费计算网页，可直接部署到 GitHub Pages。

## 功能

- 按国家、分区、渠道筛选价格。
- 支持实重与体积重，自动取较大值作为计费重量。
- 同一渠道多重量档位时，自动匹配不高于计费重的最高档；低于起重时按最低起重计费。
- 支持带电、带磁、笔类、木制品、纯纺织品、偏远，以及手工附加费。
- 输出总费用、平均每 kg、参考时效和附加费说明。
- 支持导入新版 `.xlsx` / `.csv` 运费模板。导入后会清空当前浏览器里的旧价格，并使用最新数据计算。
- 支持发布级默认价格：`data/rates.json` 会作为网站默认数据源，重新部署后所有访问者都会使用新版价格。

## GitHub Pages

仓库推送到 GitHub 后，在 `Settings > Pages` 里选择：

- Source: `Deploy from a branch`
- Branch: `main`
- Folder: `/ (root)`

保存后即可得到 GitHub Pages 访问地址。

## 数据更新说明

GitHub Pages 是静态网站，没有数据库。用户在页面导入模板后，数据保存在该用户自己的浏览器里；同一个公网地址上的其他用户需要各自导入模板。

如果需要“运营人员导入一次，所有人立刻使用最新运费”，最稳妥的静态站方案是把最新模板转换成 `data/rates.json` 后重新发布。若希望不重新发布也能全员同步，则需要增加后端数据库或云端存储。
